# Flutter 企业级落地实践-导论

> 写给有 Flutter 开发经验、但在面试中面对"非业务开发"问题无从下手的人。
> 本系列抛开 UI 与业务逻辑，站在架构师/项目经理视角，梳理一个 Flutter 应用从 0 到上架全链路中，那些"你知道它存在、但说不清楚"的关键环节。

---

## 为什么要写这个系列？

大部分 Flutter 开发者的成长路径是：学 Widget → 写页面 → 接 API → 上线。这条路径让你能"把功能做出来"，但面试中高频出现的却是另一组问题：

- "你们的推送方案怎么做的？厂商通道接了没？"
- "支付回调怎么保证幂等？"
- "应用加固做了吗？怎么做的？"
- "插件开发流程是怎样的？"
- "项目架构怎么设计的？模块间怎么通信？"

这些问题有一个共同特征：**它们不是"写代码"的问题，而是"做工程"的问题。** 经验不足的开发者往往在这类问题上暴露短板——不是不会写 Flutter，而是没站在足够高的维度理解一个 App 的全生命周期。

---

## 本系列定位

| 不是 | 而是 |
|------|------|
| 教你怎么写 Widget | 教你怎么把 App 完整交付 |
| 讲 Dart 语法和 API | 讲工程决策和架构权衡 |
| 面试题背诵手册 | 面试中能展开讲 5 分钟的实战理解 |
| 只讲 Android 或只讲 iOS | 双端对齐，标注差异点 |

核心原则：**每一个主题都回答"为什么这么做"和"不这么做会怎样"，而不只是"怎么做"。**

## 知识体系全景图

从架构师视角，一个 Flutter App 的落地生命周期可以拆解为以下阶段：

**Flutter App 全生命周期**

| 规划阶段 | 搭建阶段 | 对接阶段 | 质量阶段 | 交付阶段 |
|---------|---------|---------|---------|---------|
| 项目架构 | 插件开发 | 推送对接 | 单元测试 | 上架审核 |
| 技术选型 | 混合栈 | 支付对接 | 集成测试 | 备案许可 |
| 模块划分 | 路由设计 | 广告聚合 | 性能监控 | 打包加固 |
| CI/CD | 状态管理 | 地图/定位 | 崩溃上报 | 热更新/发版 |

这五个阶段对应本系列的五大模块，每个模块包含若干专题笔记：

### 模块一：规划与架构

| 专题 | 核心问题 | 面试高频度 |
|------|---------|-----------|

| CI/CD 搭建 | 自动化构建、分发、多环境管理 | ⭐⭐⭐ |
| 多团队协作规范 | 代码规范、Git 工作流、冲突治理 | ⭐⭐⭐ |

## 阅读指南

1. **按需阅读**：每个专题独立成篇，不需要按顺序读。面试前针对薄弱环节重点突破。
2. **追问意识**：每个专题末尾附有"面试追问"一节，模拟面试官的追问链路，帮你从"知道"到"能展开"。
3. **决策视角**：始终带着"如果让我来选，我会怎么选、为什么"的心态阅读。面试官考察的不是你用过什么，而是你做技术决策时的思考过程。
4. **实践验证**：本系列内容应结合实际项目验证。纸上得来终觉浅——哪怕是写一个 Demo 跑通支付流程，也比纯看文档有效。

## 面试中的典型场景

以下是面试中常见的"非业务开发"问题模式，对应本系列的相关专题：

**场景一：全面性考察**
> "从零搭建一个 Flutter 项目，你会怎么做？"

→ 考察你对全链路的理解：架构选型 → 工程搭建 → 第三方对接 → 质量保障 → 上架交付

**场景二：深度追问**
> "你们推送怎么做的？" → "厂商通道接了吗？" → "离线消息怎么处理？" → "推送到达率怎么统计？"

→ 单个主题的纵深追问，考察能否从"接了 SDK"深入到"解决了什么问题、有什么坑"

**场景三：决策能力**
> "为什么选 GetX 不选 Bloc？" → "如果团队大了你会换吗？" → "迁移成本怎么评估？"

→ 考察技术选型的判断力和迁移意识

**场景四：合规意识**
> "App 备案做了吗？" → "隐私政策怎么处理的？" → "SDK 列表怎么管理的？"

→ 越来越多企业把合规作为硬性要求，不懂合规的开发者会被认为"没上过线"
